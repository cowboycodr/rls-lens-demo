Live failure cases

Open the scenario that matches the bug in front of you

INSERT Missing owner column

Payload omits `user_id` while `WITH CHECK` compares it to `auth.uid()`.

 UPSERT Existing row owner conflict

`ON CONFLICT DO UPDATE` fails `USING` against the old row before the new row is checked.

 Storage Wrong object path folder

Flat upload names fail policies that expect `storage.foldername(name)[1] = auth.uid()`.

 Auth Signup profile insert runs as anon

`signUp()` returns a user, but email confirmation means the profile insert has no session yet.

 Anon Public form accepts any row

`TO anon WITH CHECK (true)` fixes submission but leaves every writable column open.

 Authenticated Every signed-in user passes

`USING (true)` and `WITH CHECK (true)` move all row isolation back into app code.
